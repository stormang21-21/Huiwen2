- **❌ Upload failed:** POST to `/api/media/upload` returned 404 - endpoint doesn't exist

### Root Cause Analysis
After researching OpenClaw documentation:
- OpenClaw Gateway does NOT have `/api/media/upload` REST endpoint
- Proper API architecture is:
  1. **WebSocket API** (`chat.send`, `chat.history`) for Control UI
  2. **OpenAI-compatible HTTP API** (`/v1/chat/completions`) for third-party apps
  3. **No direct media upload endpoint** - media must go through message pipeline

### Solution Architecture (Identified)
**Proper approach for standalone voice chat web page:**
1. Enable OpenAI-compatible HTTP endpoint: `gateway.http.endpoints.chatCompletions.enabled: true`
2. Connect page to WebSocket for real-time chat (`chat.send`, `chat.history`)
3. For voice input: Transcribe locally with OpenAI Whisper API (user's API key)
4. For voice output: Use TTS directly or let OpenClaw handle it via `[[tts]]` tags
5. Session routing: Use `x-openclaw-session-key` header for consistent sessions

### OpenClaw API Endpoints (Documentation Reference)
- **WebSocket:** `ws://gateway:18789` - methods: `chat.history`, `chat.send`, `chat.abort`, `chat.inject`
- **HTTP (when enabled):**
  - `GET /v1/models` - list agents
  - `POST /v1/chat/completions` - OpenAI-compatible chat
  - `POST /v1/embeddings` - embeddings
  - `POST /v1/responses` - agent-native responses
- **Auth:** Gateway token required (`Authorization: Bearer <token>`)
